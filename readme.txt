задачи приложения которое пишется - идентификация обьектов и их размещения на складе, а так же их эффективное манипулирование, изменение и перемещение.
Предположим большое складское помещение в котором есть большие Шкафы на которых есть Полки.На полках будут размещаться некие Боксы в которых будет содержаться какое то количество товаров. Помимо этого на каждом шкафу будут подвесные Цепи, на которые так же будут размещаться Боксы но уже только для одного наименования товара.
Клиентское приложение должно уметь подключаться по сети к серверному приложению, и через авторизацию получить доступ к разрешенным функциям (например для админов -  добавить новую сущность(товар),модифицировать характеристики, переместитьна другой Шкаф(на котором допустим храним обьекты которые хотим списать потому что ждем когда клиент заберет оптовую партию по предварительному заказу) или для простых пользователей которые могут только просматривать и ну допустим менять только количество имеющихся экземпляров продукции(при продаже или списании)).
